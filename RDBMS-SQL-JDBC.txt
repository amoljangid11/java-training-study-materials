Introduction to RDBMS: 1. What is a Database?

A database is an organized collection of data stored electronically so that it can be easily accessed, managed, and updated.

Real-Life Examples

Banking systems

Hospital systems

Railway reservation systems

Example

2. What is RDBMS?

RDBMS = Relational Database Management System 

An RDBMS is software used to create, manage, and manipulate relational databases.

It stores data in the form of tables.

3. Why is it called “Relational”?

Because tables are related to each other using relationships.

Example:

One student can enroll in many courses
One department can have many employees

These relationships are managed using keys. Foreign Key

5. Features of RDBMS

a) Table-Based Storage

Data is stored in rows and columns.

b) Relationships

c) Data Integrity

d) Security

6. Basic Terminologies in RDBMS

7. Types of Relationships in RDBMS

a) One-to-One (1:1)

One person → One passport

b) One-to-Many (1:M)

One department → Many employees

c) Many-to-Many (M:M)

Many students → Many courses

This requires a junction table.

11. What is a Transaction?

A transaction is a logical unit of work.

Example:

Bank money transfer:

Debit from Account A
Credit to Account B

Both operations must succeed together.

12. ACID Properties

ACID ensures reliable transactions.

A → Atomicity

Either complete fully or not at all.

C → Consistency

Database remains valid.

I → Isolation

Transactions do not interfere.

D → Durability

Committed data remains permanent.

Select Statement : The SELECT statement is used to retrieve data from one or more tables in a MySQL database.

It is one of the most important SQL commands.

Basic Syntax

SELECT column1, column2, ...
FROM table_name;

OR

SELECT * FROM table_name;

SELECT → specifies columns to retrieve
FROM → specifies table name
* → selects all columns

INNER JOIN Example : 

SELECT e.emp_name,
       e.salary,
       d.dept_name
FROM employees e
INNER JOIN departments d
ON e.department = d.dept_name;

LEFT JOIN Example :

select e.emp_id, e.emp_name, d.dept_name from employees e left join departments d on e.department = d.dept_name;

RIGHT JOIN Example :

select e.emp_id, e.emp_name, d.dept_name from employees e right join departments d on e.department = d.dept_name;

FULL OUTER JOIN : 


select e.emp_id, e.emp_name, d.dept_name from employees e left join departments d on e.department = d.dept_name union select e.emp_id, e.emp_name, d.dept_name from employees e right join departments d on e.department = d.dept_name;

CROSS JOIN :

select e.emp_name, d.dept_name from employees e cross join departments d;

SELF JOIN : 

select e.emp_name, m.salary from employees e left join employees m on e.emp_id = m.emp_id;

2. ALTER Command

The ALTER command modifies an existing table structure.

Example 1: Add New Column

Add email column:

ALTER TABLE employees
ADD email VARCHAR(100);

Example 2: Modify Column Data Type

Increase department size:

ALTER TABLE employees
MODIFY department VARCHAR(100);

Example 3: Rename Column

Rename emp_name to employee_name:

ALTER TABLE employees
RENAME COLUMN emp_name TO employee_name;

Example 4: Drop Column

Remove email column:

ALTER TABLE employees
DROP COLUMN email;

3. DROP Command

The DROP command permanently deletes database objects.

Example 1: Drop Table

DROP TABLE employees;

⚠️ Entire table structure and data will be deleted permanently.

Example 2: Drop Database

DROP DATABASE company_db;

⚠️ Entire database will be removed permanently.

4. TRUNCATE Command

TRUNCATE removes all rows from a table but keeps the table structure.

Example

TRUNCATE TABLE employees;

Difference Between DELETE and TRUNCATE

Feature	DELETE	TRUNCATE
Removes specific rows	Yes	No
WHERE clause allowed	Yes	No
Removes all rows	Optional	Yes
Faster	No	Yes
Resets AUTO_INCREMENT	No	Yes

5. RENAME Command

Used to rename tables.

Example

RENAME TABLE employees TO staff_members;
