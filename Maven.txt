What is Maven?

Apache Maven is a powerful build automation and project management tool primarily used for Java applications.

It helps developers to:

Manage project dependencies (JAR files)
Compile source code
Run unit tests
Package applications (JAR/WAR)
Deploy applications
Maintain a standard project structure

Maven simplifies the development process by using a configuration file called:

pom.xml

POM stands for:

Project Object Model

Why Maven is Used?

Before Maven, developers had to manually:

Download JAR files
Add libraries to classpath
Compile code manually
Maintain project builds

Maven automates all these tasks.

Introduction to Maven

Apache Maven is a powerful build automation and project management tool mainly used for Java projects. It simplifies project setup, dependency management, compilation, testing, packaging, and deployment.

Maven follows the concept of:

Convention over Configuration
Standardized project structure
Dependency management through repositories
Automated build lifecycle

1. Dependency Management

Maven automatically downloads required libraries (JAR files) from repositories.

Example:

An archetype is a Maven project template used to generate a new project structure automatically.

Generate Maven Project Using Archetype
Command
mvn archetype:generate
Important Parameters
Parameter	Description
groupId	Company/project package
artifactId	Project name
version	Project version
package	Base package name
Example: Create Java Project
mvn archetype:generate ^
-DgroupId=com.example ^
-DartifactId=employee-app ^
-DarchetypeArtifactId=maven-archetype-quickstart ^
-DinteractiveMode=false

Maven Build Lifecycle

Maven lifecycle defines the sequence of build phases.

There are 3 built-in lifecycles:

Lifecycle	Purpose
Default	Build and deployment
Clean	Cleanup
Site	Documentation
1. Default Lifecycle

This is the main lifecycle.

Important Phases
Phase	Description
validate	Validate project
compile	Compile source code
test	Run unit tests
package	Create JAR/WAR
verify	Run quality checks
install	Install into local repository
deploy	Deploy to remote repository
Maven Lifecycle Flow
validate
   ↓
compile
   ↓
test
   ↓
package
   ↓
verify
   ↓
install
   ↓
deploy
Example Commands
Compile Project
mvn compile
Run Tests
mvn test
Create JAR File
mvn package
Install to Local Repository
mvn install
Deploy to Remote Repository
mvn deploy
Important Concept

If you run:

mvn package

Maven executes all previous phases automatically:

validate → compile → test → package
2. Clean Lifecycle

Used to remove generated files.

Phases

Step 3: Java JDBC Program
EmployeeInsert.java
package com.example;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;

public class EmployeeInsert {

    public static void main(String[] args) {

        String url = "jdbc:mysql://localhost:3306/company";
        String username = "root";
        String password = "root";

        try {

            // Create connection
            Connection con = DriverManager.getConnection(
                    url,
                    username,
                    password);

            // SQL Query
            String sql =
                    "INSERT INTO employee(id,name) VALUES(?,?)";

            PreparedStatement ps =
                    con.prepareStatement(sql);

            ps.setInt(1, 101);
            ps.setString(2, "Amol");

            int rows = ps.executeUpdate();

            System.out.println(rows + " row inserted");

            con.close();

        } catch (Exception e) {

            e.printStackTrace();

        }
    }
}
