Junit/Code Coverage : Introduction to Junit : What is JUnit?

JUnit is a popular unit testing framework for Java applications.
It is used to test individual units or components of a program automatically.

A unit usually means:

A method
A class
A small functionality of the application

JUnit helps developers:

Find bugs early
Improve code quality
Automate testing
Support Test-Driven Development (TDD) // first we write the test case after that we write the code

Why Use JUnit?

Without JUnit:

Developers manually test the application
Testing becomes repetitive
Bugs are missed easily

With JUnit:

Tests run automatically
Faster debugging
Safer code refactoring
Easy integration with Maven, Gradle, Jenkins, CI/CD

Features of JUnit

1. Automated Testing

Run tests automatically without manual checking. // true

2. Assertions

Verify expected and actual outputs.

3. Test Suites

Group multiple test cases together.

4. Annotations

Control test execution using annotations like: JF code testing

@Test  used to write the test cases
@BeforeEach  // how was the code before execution of test cases
@AfterEach  // after test cases execution

5. Integration Support

Works with:

Maven
Gradle
Spring Boot
Jenkins
IDEs like IntelliJ and Eclipse

JUnit Versions

Version	Description
JUnit 3	Old version
JUnit 4	Annotation-based
JUnit 5	Latest and most powerful

JUnit 5 is recommended today.

Architecture of JUnit 5

JUnit 5 consists of:

Component	Purpose
JUnit Platform	Launches testing frameworks
JUnit Jupiter	New programming model
JUnit Vintage	Supports old JUnit tests

11. Complete JUnit Example

CalculatorTest.java

package com.demo;

import static org.junit.jupiter.api.Assertions.*;

import org.junit.jupiter.api.AfterAll;
import org.junit.jupiter.api.AfterEach;
import org.junit.jupiter.api.BeforeAll;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Disabled;
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    Calculator calculator;

    // Runs once before all tests

    @BeforeAll
    static void beforeAllTests() {
        System.out.println("=== Starting Test Execution ===");
    }

    // Runs before each test
    @BeforeEach
    void setup() {
        calculator = new Calculator();
        System.out.println("Setup completed");
    }

    // Test method
    @Test
    @DisplayName("Addition Test")
    void testAdd() {

        int expected = 30;
        int actual = calculator.add(10, 20);

        assertEquals(expected, actual);
    }

    @Test
    @DisplayName("Subtraction Test")
    void testSubtract() {

        int result = calculator.subtract(50, 20);

        assertEquals(30, result);
    }

    @Test
    @DisplayName("Multiplication Test")
    void testMultiply() {

        int result = calculator.multiply(5, 4);

        assertEquals(20, result);
    }

    @Test
    @DisplayName("Division Test")
    void testDivide() {

        int result = calculator.divide(20, 5);

        assertEquals(4, result);
    }

    @Test
    @DisplayName("Even Number Test")
    void testEven() {

        boolean result = calculator.isEven(10);

        assertTrue(result);
    }

    @Test
    @DisplayName("Message Test")
    void testMessage() {

        String msg = calculator.getMessage();

        assertNotNull(msg);

        assertEquals("Welcome to JUnit", msg);
    }

    @Test
    @DisplayName("Assert False Example")
    void testFalseCondition() {

        boolean result = calculator.isEven(9);

        assertFalse(result);
    }

    @Test
    @DisplayName("Assert Null Example")
    void testNull() {

        String value = null;

        assertNull(value);
    }

    @Test
    @DisplayName("Assert Throws Example")
    void testException() {

        assertThrows(ArithmeticException.class, () -> {
            calculator.divide(10, 0);
        });
    }

    @Disabled
    @Test
    @DisplayName("Disabled Test Example")
    void disabledTest() {

        fail("This test is disabled");
    }

    // Runs after each test
    @AfterEach
    void tearDown() {
        System.out.println("Test completed");
    }

    // Runs once after all tests
    @AfterAll
    static void afterAllTests() {
        System.out.println("=== Test Execution Completed ===");
    }
}

12. Understanding Assert Methods

assertEquals()

Checks expected and actual values are equal.

assertEquals(10, result);

Example:

assertEquals(30, calculator.add(10, 20));
assertNotEquals()

Checks values are NOT equal.

assertNotEquals(100, result);

assertTrue()

Checks condition is true.

assertTrue(result);

Example:

assertTrue(calculator.isEven(10));

assertFalse()

Checks condition is false.

assertFalse(result);

assertNull()

Checks object is null.

assertNull(value);

assertNotNull()

Checks object is NOT null.

assertNotNull(value);

assertThrows()

Checks exception occurs.

assertThrows(ArithmeticException.class, () -> {
    calculator.divide(10, 0);
});

fail()

Forcefully fails test.

fail("Test failed");

JUnit lifecycle annotations:

Annotation	Purpose

@AfterAll 	Runs once after all tests

@AfterEach	Runs after every test

1️⃣6️⃣ Lifecycle Example

package com.demo.test;

import org.junit.jupiter.api.*;

public class DemoLifecycleTest {

    @BeforeAll
    static void beforeAll() {
        System.out.println("Before All Tests");
    }

    @BeforeEach
    void beforeEach() {
        System.out.println("Before Each Test");
    }

    @Test
    void test1() {
        System.out.println("Test 1");
    }

    @Test
    void test2() {
        System.out.println("Test 2");
    }

    @AfterEach
    void afterEach() {
        System.out.println("After Each Test");
    }

    @AfterAll
    static void afterAll() {
        System.out.println("After All Tests");
    }
}
