Java Fundamentals : Java Architecture : Java architecture is designed to achieve one core goal: “Write Once, Run Anywhere” (WORA). Let’s break it down from scratch in a clear, layered way so you can build a strong foundation.

🔷 1. High-Level Java Architecture

At a high level, Java architecture consists of:

Java Source Code (.java)
        ↓
Java Compiler (javac)
        ↓
Bytecode (.class)
        ↓
Java Virtual Machine (JVM)  binary code
        ↓
Operating System / Hardware

🔷 2. Step-by-Step Flow

✅ Step 1: Writing Code

You write Java code in a .java file.

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}
✅ Step 2: Compilation (Javac)

The Java compiler (javac) converts source code into Bytecode.

👉 Output:

Hello.class

✔ Bytecode is platform-independent WORE

✅ Step 3: Execution (JVM)

The Java Virtual Machine (JVM) executes the bytecode.

✔ JVM is platform-dependent
✔ It converts bytecode into machine code binary code 01

🔷 3. Core Components of Java Architecture

🧩 A. JVM (Java Virtual Machine)

The JVM is the heart of Java.

🔹 Internal Components:

1. Class Loader Subsystem

Loads .class files into memory.

Types:

Bootstrap ClassLoader
Extension ClassLoader
Application ClassLoader

2. Runtime Data Areas (Memory Structure)

JVM memory is divided into:

📦 Method Area

Stores class metadata, static variables, methods

📦 Heap

Stores objects
Shared across threads

📦 Stack

Stores method calls, local variables
One per thread

📦 PC Register

Holds current instruction address

📦 Native Method Stack

Handles native (C/C++) methods

3. Execution Engine

Executes bytecode using:

🔸 Interpreter

Executes line by line (slower)

🔸 JIT Compiler (Just-In-Time)

Converts bytecode into native machine code
Improves performance

4. Garbage Collector (GC)

Automatically removes unused objects from heap
Prevents memory leaks

🧩 B. JRE (Java Runtime Environment)

JRE = JVM + Libraries

Includes:

JVM
Core libraries (java.lang, java.util, etc.)

👉 Used to run Java applications

🧩 C. JDK (Java Development Kit)

JDK = JRE + Development Tools

Includes:

javac (compiler)
java (runtime launcher)
jdb (debugger)
jar (archiver)

👉 Used to develop Java applications

Flow Control Statements - ForEach : 🔁 Flow Control Statement in Java: forEach (From Scratch)

In Java, forEach is a modern iteration mechanism introduced in Java 8. It is mainly used to iterate over collections (like List, Set) and arrays in a cleaner and more readable way compared to traditional loops.

🧠 1. What is forEach?

forEach is a method defined in the Iterable interface and also available in Streams.

👉 It takes a lambda expression (or method reference) and applies it to each element.

⚙️ 2. Syntax
collection.forEach(element -> {
    // logic
});

OR (Method Reference)

collection.forEach(System.out::println);

🧩 3. Basic Example with List

import java.util.*; // LinkedList, Set, HashSet, Map, HashMap collection classes

public class ForEachExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Amol", "Rahul", "Sneha");

        names.forEach(name -> {
            System.out.println(name);
        });
    }
}

👉 Output:

Amol
Rahul
Sneha

⚡ 4. Using Method Reference (Cleaner Way)

names.forEach(System.out::println);

✔ This is equivalent to the previous example but more concise.

Arrays - Parallel Array Sorting : 🔷 What is an Array in Java?
An array in Java is a fixed-size data structure used to store multiple values of the same type in a single variable.

int[] marks = {85, 90, 78, 92};


Index-based (starts from 0)


Fixed size


Same data type



🔷 What is Parallel Array?

A parallel array means multiple arrays that are related by index.

👉 Example:

String[] names = {"Amol", "Ravi", "Neha"};   int[] marks = {85, 90, 78};

Here:


names[0] → Amol


marks[0] → 85


Both represent same student data

🔷 Problem Statement

👉 If you sort only one array (e.g., marks), data gets mismatched:

Arrays.sort(marks);

Now:
marks = {78, 85, 90} names = {"Amol", "Ravi", "Neha"} ❌ WRONG mapping

🔷 What is Parallel Array Sorting?

Parallel array sorting means:

👉 When sorting one array, we rearrange other arrays accordingly to maintain relationships.

🔶 Example 1: Sort Students by Marks (Ascending)

Input:

String[] names = {"Amol", "Ravi", "Neha"};   int[] marks = {85, 90, 78};

Solution (Bubble Sort Approach) :

public class ParallelArraySort {    

public static void main(String[] args) {       

 String[] names = {"Amol", "Ravi", "Neha"};       
 int[] marks = {85, 90, 78};       

 int n = marks.length;        

for (int i = 0; i < n - 1; i++) {        

    for (int j = 0; j < n - i - 1; j++) {           

     if (marks[j] > marks[j + 1]) {         

           // Swap marks                   

 int tempMark = marks[j];                    
marks[j] = marks[j + 1];                   
 marks[j + 1] = tempMark;                    

// Swap names accordingly                  

  String tempName = names[j];                    

names[j] = names[j + 1];                   

 names[j + 1] = tempName;                

}            

}       

 }        // Output       

 for (int i = 0; i < n; i++)  {            

System.out.println(names[i] + " - " + marks[i]);        

}    
}
}


Output:

Neha - 78Amol - 85Ravi - 90

 int temp = marks[j];

 // swap names

 grades[j] = grades[j + 1];

}

}

🔥 Real-World Combined Example (OOP + Threads + Exception)

class BankAccount {
    private int balance = 1000; // Encapsulation

    synchronized void withdraw(int amount) {
        if(balance >= amount) {
            balance -= amount;
            System.out.println("Withdrawn: " + amount);
        } else {
            System.out.println("Insufficient Balance");
        }
    }
}

class UserThread extends Thread {
    BankAccount acc;

    UserThread(BankAccount acc) {
        this.acc = acc;
    }

    public void run() {
        acc.withdraw(700);
    }
}

public class BankingApp {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount();

        UserThread t1 = new UserThread(acc);
        UserThread t2 = new UserThread(acc);

        t1.start();
        t2.start();
    }
}
