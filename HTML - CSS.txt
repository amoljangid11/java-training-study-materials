HTML Templates : HTML Templates Explained from Scratch

HTML <template> is a special HTML element used to store reusable HTML content that is not rendered immediately on the webpage.

It is useful when you want to:

Dynamically create UI components
Reuse HTML blocks
Improve performance
Build cards, tables, modals, lists, etc.
Work with JavaScript frameworks and Web Components
1. What is an HTML Template?

The <template> tag contains HTML code that is hidden until JavaScript uses it.

Syntax
<template id="templateName">
    <!-- Hidden HTML -->
</template>

The content inside:

Does NOT display on the page initially
Does NOT execute scripts immediately
Can be cloned and inserted using JavaScript
2. Why Use Templates?

Without templates:

<div class="card">
   <h2>John</h2>
   <p>Developer</p>
</div>

<div class="card">
   <h2>Mike</h2>
   <p>Tester</p>
</div>

This creates repeated code.

With templates:

Write once
Reuse many times
Dynamically inject data
3. Basic HTML Template Example
Example
<!DOCTYPE html>
<html>
<head>
    <title>HTML Template Example</title>
</head>
<body>

<h1>HTML Template Demo</h1>

<template id="myTemplate">
    <div class="card">
        <h2>Name</h2>
        <p>Role</p>
    </div>
</template>

<script>

    // Access template
    const template = document.getElementById("myTemplate");

    // Clone template content
    const clone = template.content.cloneNode(true);

    // Add clone to body
    document.body.appendChild(clone);

</script>

</body>
</html>
4. Understanding Key Concepts
template.content

Returns the hidden DOM inside the template.

template.content
cloneNode(true)

Creates a deep copy of the template.

cloneNode(true)
true means:
Copy all child elements
false means:
Copy only parent node
5. Dynamic Template Example

Create employee cards dynamically.

Complete Example
<!DOCTYPE html>
<html>
<head>
    <title>Employee Cards</title>

    <style>

        .card{
            border:1px solid gray;
            padding:15px;
            margin:10px;
            width:250px;
            border-radius:10px;
            background:#f5f5f5;
        }

    </style>

</head>
<body>

<h1>Employee List</h1>

<div id="container"></div>

<!-- Template -->
<template id="employeeTemplate">

    <div class="card">

        <h2 class="name"></h2>

        <p class="designation"></p>

        <button>View Profile</button>

    </div>

</template>

<script>

    const employees = [

        {
            name:"Amol",
            designation:"Java Developer"
        },

        {
            name:"Ravi",
            designation:"Angular Developer"
        },

        {
            name:"Sneha",
            designation:"Python Developer"
        }

    ];

    const template = document.getElementById("employeeTemplate");

    const container = document.getElementById("container");

    employees.forEach(emp => {

        // Clone template
        const clone = template.content.cloneNode(true);

        // Modify cloned content
        clone.querySelector(".name").textContent = emp.name;

        clone.querySelector(".designation").textContent =
            emp.designation;

        // Append to container
        container.appendChild(clone);

    });

</script>

</body>
</html>

Semantic HTML : Semantic HTML — Complete Beginner to Advanced Guide

Semantic HTML means using HTML tags according to their meaning rather than only for styling or layout.

Semantic elements clearly describe:

The purpose of the content
The structure of the webpage
The relationship between sections

This helps:

Browsers
Search engines (SEO)
Screen readers
Developers
Accessibility tools
1. What is Semantic HTML?
Non-Semantic Example
<div class="header">
    <div class="menu">
        <div class="menu-item">Home</div>
        <div class="menu-item">About</div>
    </div>
</div>

<div class="content">
    <div class="article">
        Welcome to our website
    </div>
</div>

<div class="footer">
    Copyright 2026
</div>

Problem:

Everything is <div>
Browser does not understand meaning
Poor accessibility
Difficult maintenance
Semantic Example
<header>
    <nav>
        <a href="#">Home</a>
        <a href="#">About</a>
    </nav>
</header>

<main>
    <article>
        Welcome to our website
    </article>
</main>

<footer>
    Copyright 2026
</footer>

Now:

<header> = top section
<nav> = navigation
<main> = main content
<article> = independent content
<footer> = bottom section

This is semantic HTML.

2. Benefits of Semantic HTML
Benefit	Description
Readability	Easier to understand code
SEO	Search engines understand page structure
Accessibility	Better support for screen readers
Maintenance	Easier future updates
Reusability	Better component structure
Browser Understanding	Browser knows page meaning

What is Geolocation?

Geolocation API allows browsers to access user location.

Geolocation Example
<!DOCTYPE html>
<html>
<head>
    <title>Geolocation Example</title>
</head>

<body>

<h1>Get Current Location</h1>

<button onclick="getLocation()">
    Get Location
</button>

<p id="result"></p>

<script>

function getLocation() {

    if (navigator.geolocation) {

        navigator.geolocation.getCurrentPosition(
            showPosition,
            showError
        );

    } else {
        document.getElementById("result").innerHTML =
            "Geolocation not supported";
    }
}

function showPosition(position) {

    document.getElementById("result").innerHTML =
        "Latitude: " + position.coords.latitude +
        "<br>Longitude: " + position.coords.longitude;
}

function showError(error) {

    switch(error.code) {

        case error.PERMISSION_DENIED:
            alert("Permission denied");
            break;

        case error.POSITION_UNAVAILABLE:
            alert("Location unavailable");
            break;

        case error.TIMEOUT:
            alert("Request timeout");
            break;

        default:
            alert("Unknown error");
    }
}

</script>

</body>
</html>

Total Width Calculation

Total Width =
Content Width
+ Left Padding + Right Padding
+ Left Border + Right Border
+ Left Margin + Right Margin

300
+ 20 + 20
+ 5 + 5
+ 40 + 40

= 430px
