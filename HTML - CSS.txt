HTML Templates : HTML <template> is a special HTML element used to store reusable HTML content that is not rendered immediately on the webpage.

It is useful when you want to:

Dynamically create UI components
Reuse HTML blocks
Improve performance
Build cards, tables, modals, lists, etc.
Work with JavaScript frameworks and Web Components

1. What is an HTML Template?

The <template> tag contains HTML code that is hidden until JavaScript uses it.

Syntax
<template id="templateName">
    <!-- Hidden HTML -->
</template>

The content inside:

Does NOT display on the page initially
Does NOT execute scripts immediately
Can be cloned and inserted using JavaScript

2. Why Use Templates?

Without templates:

<div class="card">
   <h2>John</h2>
   <p>Developer</p> CSS
</div>

<div class="card">
   <h2>Mike</h2>
   <p>Tester</p>
</div>

This creates repeated code.

With templates:

Write once
Reuse many times
Dynamically inject data

3. Basic HTML Template Example

Example

<!DOCTYPE html>
<html>
<head>
    <title>HTML Template Example</title>
</head>
<body>

<h1>HTML Template Demo</h1>

<template id="myTemplate">
    <div class="card">
        <h2>Name</h2>
        <p>Role</p>
    </div>
</template>

<script>

    // Access template
    const template = document.getElementById("myTemplate");

    // Clone template content
    const clone = template.content.cloneNode(true);

    // Add clone to body
    document.body.appendChild(clone);

</script>

</body>
</html>

4. Understanding Key Concepts

template.content

Returns the hidden DOM inside the template.

template.content

cloneNode(true)

Creates a deep copy of the template.

cloneNode(true)

true means:
Copy all child elements
false means:

Copy only parent node

5. Dynamic Template Example

Create employee cards dynamically. //DOM

Complete Example
<!DOCTYPE html>
<html>
<head>
    <title>Employee Cards</title>

    <style>

        .card{
            border:1px solid gray;
            padding:15px;
            margin:10px;
            width:250px;
            border-radius:10px;
            background:#f5f5f5;
        }

    </style>

</head>
<body>

<h1>Employee List</h1>

<div id="container"></div>

<!-- Template -->
<template id="employeeTemplate">

    <div class="card">

        <h2 class="name"></h2>

        <p class="designation"></p>

        <button>View Profile</button>

    </div>

</template>

<script>

    const employees = [ // Array of Object

        {
            name:"Amol",
            designation:"Java Developer"
        },

        {
            name:"Ravi",
            designation:"Angular Developer"
        },

        {
            name:"Sneha",
            designation:"Python Developer"
        }

    ];

    const template = document.getElementById("employeeTemplate");

    const container = document.getElementById("container");

    employees.forEach(emp => {

        // Clone template
        const clone = template.content.cloneNode(true);

        // Modify cloned content
        clone.querySelector(".name").textContent = emp.name;

        clone.querySelector(".designation").textContent =
            emp.designation;

        // Append to container
        container.appendChild(clone);

    });

</script>

</body>
</html>

7. Template vs Hidden Div

Using Hidden Div

<div id="temp" style="display:none">
</div>

Problems:

Added to DOM immediately
Less efficient
CSS/JS may affect it

Using Template

<template>
</template>

Advantages:

Not rendered
Cleaner
Better performance
Safer DOM handling

1. Semantic HTML : HTML5 header, section, article, footer, aside, details, summary, nav

What is Semantic HTML?

Semantic HTML means using HTML tags according to their meaning.

Instead of using only <div> everywhere, semantic elements describe the purpose of content.

Benefits :

Better SEO
Better Accessibility
Easier maintenance
Better readability
Helps screen readers // those who are not able to access the contents of HTML like blind person

Common Semantic Tags

Tag	Purpose

<header>	Top section
<nav>	Navigation links
<main>	Main content
<section>	Grouped content   <div>
<article>	Independent content
<aside>	Sidebar
<footer>	Bottom section
<figure>	Images/diagrams
<figcaption>	Caption

4. Geolocation API

What is Geolocation?

Geolocation API allows browsers to access user location.

Uses :

Maps
Delivery apps
Tracking
Nearby services

Important Notes :

Requires user permission

Works better with HTTPS

Geolocation Example :

<!DOCTYPE html>
<html>
<head>
    <title>Geolocation Example</title>
</head>

<body>

<h1>Get Current Location</h1>

<button onclick="getLocation()">
    Get Location
</button>

<p id="result"></p>

<script>

function getLocation() {

    if (navigator.geolocation) {

        navigator.geolocation.getCurrentPosition(
            showPosition,
            showError
        );

    } else {
        document.getElementById("result").innerHTML =
            "Geolocation not supported";
    }
}

function showPosition(position) {

    document.getElementById("result").innerHTML =
        "Latitude: " + position.coords.latitude +
        "<br>Longitude: " + position.coords.longitude;
}

function showError(error) {

    switch(error.code) {

        case error.PERMISSION_DENIED:
            alert("Permission denied");
            break;

        case error.POSITION_UNAVAILABLE:
            alert("Location unavailable");
            break;

        case error.TIMEOUT:
            alert("Request timeout");
            break;

        default:
            alert("Unknown error");
    }
}

</script>

</body>
</html>

5. Web Storage API

What is Web Storage?

HTML5 provides client-side storage.

Types

Storage	Lifetime

localStorage	Permanent
sessionStorage 	Until tab closes JavaScript predefined

localStorage Example

<!DOCTYPE html>
<html>
<head>
    <title>Local Storage</title>
</head>

<body>

<h1>Local Storage Example</h1>

<input type="text" id="username">

<button onclick="saveData()">
    Save
</button>

<button onclick="getData()">
    Get
</button>

<p id="output"></p>

<script>

function saveData() {

    let name =
        document.getElementById("username").value;

    localStorage.setItem("username", name);

    alert("Data Saved");
}

function getData() {

    let name =
        localStorage.getItem("username");

    document.getElementById("output").innerHTML =
        "Stored Name: " + name;
}

</script>

</body>
</html>

sessionStorage Example

    sessionStorage.getItem("course");
